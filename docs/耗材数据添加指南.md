## 1. 快速开始：添加一个新耗材

添加一个新耗材的完整流程：

```
厂商数据表（PDF/网页）
    ↓ 分析属性，映射到 canonical ID
    ↓ 有新属性？→ 更新 registry + labels
    ↓
创建 JSON 文件 → npm run build → 验证
```

**你只需要改动的文件：**

| 场景 | 需要改的文件 |
|---|---|
| 所有属性都已存在 | 只创建 1 个 JSON 文件 |
| 有新属性需要添加 | JSON + `spec-registry.ts` + `spec-labels.ts` |

**不需要改的文件：** 页面组件、对比页面、CompareWidget —— 它们会自动适配新数据。

## 2. Step 1：创建 JSON 数据文件

在 `src/content/filaments/` 下创建新文件，文件名用 `品牌-型号.json` 格式，全小写，用连字符分隔。

**示例：添加 Creality Hyper PLA**

文件路径：`src/content/filaments/creality-hyper-pla.json`

```json
{
  "brand": "Creality",
  "model": "Hyper PLA",
  "category": "pla",
  "image": "https://example.com/creality-hyper-pla.jpg",
  "priceRange": "¥59-79/卷",
  "officialUrl": "https://www.creality.com/products/hyper-pla",
  "affiliateUrl": "",
  "properties": {
    "material_type": {
      "value": "PLA (Polylactic Acid)",
      "value_zh": "PLA（聚乳酸）"
    },
    "diameter": {
      "value": 1.75,
      "unit": "mm"
    },
    "density": {
      "value": 1.24,
      "unit": "g/cm³"
    },
    "tensile_strength_xy": {
      "value": 52.0,
      "unit": "MPa"
    }
    // ... 更多属性
  },
  "printing_guidelines": {
    "nozzle_temperature_celsius": {
      "range": [190, 230],
      "recommended": 210
    },
    "bed_temperature_celsius": [25, 60],
    "fan_speed_percent": 100,
    "print_speed_mm_s": "200-600",
    "heated_bed_required": false,
    "note": {
      "zh-cn": "高速打印专用耗材",
      "en": "High-speed printing filament"
    }
  },
  "keywords": {
    "zh-cn": ["PLA", "Creality", "高速", "Hyper"],
    "en": ["PLA", "Creality", "High Speed", "Hyper"]
  }
}
```

### JSON 结构说明

| 字段 | 必填 | 说明 |
|---|---|---|
| `brand` | 是 | 品牌名，如 "Creality" |
| `model` | 是 | 产品型号，如 "Hyper PLA" |
| `category` | 是 | 材料大类：`pla` / `petg` / `abs` / `tpu` / `nylon` / `pc` / `asa` / `other` |
| `image` | 否 | 产品图片 URL |
| `priceRange` | 否 | 价格区间字符串 |
| `officialUrl` | 否 | 官方产品页 |
| `affiliateUrl` | 否 | 联盟推广链接 |
| `properties` | 否 | 结构化属性数据（核心，见下文） |
| `printing_guidelines` | 否 | 打印参数 |
| `keywords` | 否 | 多语言搜索关键词 |

## 3. Step 2：处理属性映射（核心）

当你拿到一个新厂商的数据表时，**第一个问题是：这些属性在系统中是否已经存在？**

### 判断流程

```
拿到厂商数据表中的一个属性名
    ↓
在「属性映射速查表」中查找
    ↓
找到了 → 使用对应的 canonical ID，不需要改代码
没找到 → 这是新属性，需要添加（见 Step 3）
```

### ⚠️ 最高原则：只填官方数据，禁止编造

**绝对不要编造（YY）任何数值。** 所有填入 JSON 的数据必须有明确的官方来源（厂商官网、产品说明书、TDS 数据表）。

- ✅ 厂商官网明确标注的参数 → 填写
- ✅ 厂商提供的 PDF/TDS 数据表中的参数 → 填写
- ❌ "大概是这个范围吧"、"PLA 一般都是 xxx" → **不填**
- ❌ 参考其他品牌凑出来的数值 → **不填**
- ❌ 用 AI 生成的估计值 → **不填**

**没有官方数据就留空，不要填。** 对比页面会自动处理缺失情况——缺少的厂商显示 "—"。

### 属性值的填写规则

#### 规则 1：纯数值属性 → `value` 用数字

```json
"tensile_strength_xy": {
  "value": 52.0,
  "unit": "MPa"
}
```

✅ `value` 是 `number` 类型 → 前端可以直接做数值对比、柱状图

❌ 不要写成 `"value": "52.0 MPa"` → 失去数值对比能力

#### 规则 2：范围值 → `value` 用字符串

```json
"heat_deflection_temperature": {
  "value": "50-60",
  "unit": "°C",
  "condition": "0.45 MPa"
}
```

#### 规则 3：定性属性（如"容易"、"低"）→ `value` 用英文 + `value_zh` 中文

```json
"printability": {
  "value": "Easy",
  "value_zh": "容易"
}
```

#### 规则 4：品牌特有描述 → `value` 英文 + `value_zh` 中文

```json
"advantage": {
  "value": "High-speed printing optimized",
  "value_zh": "专为高速打印优化"
}
```

#### 可选字段

| 字段 | 何时使用 | 示例 |
|---|---|---|
| `unit` | 有物理单位时 | `"MPa"`, `"°C"`, `"%"`, `"g/cm³"` |
| `standard` | 有测试标准时 | `"ISO 527"`, `"ASTM D256"` |
| `test_method` | 同一属性有不同测试方法时 | `"notched_izod"`, `"charpy"` |
| `condition` | 有测试条件或测量不确定度时 | `"0.45 MPa"`, `"± 3.9"` |

> 💡 **关于 ± 值：** 如果厂商给出的数据包含标准差（如 `15.7 ± 3.9`），把主值填入 `value`，把 ± 部分填入 `condition`：
>
> ```json
> "toughness": {
>   "value": 15.7,
>   "unit": "kJ/㎡",
>   "condition": "± 3.9"
> }
> ```

## 4. Step 3：更新 spec-registry.ts（仅新属性时需要）

**文件：** `src/content/filaments/spec-registry.ts`

如果厂商的数据表中有**全新的属性**（在速查表中找不到），你需要：

### 4.1 添加 canonical ID

在 `FilamentSpecId` 类型中添加新的 ID，使用 `snake_case` 命名：

```typescript
export type FilamentSpecId =
  // ... 已有属性
  | '新的属性_id';      // ← 添加这行
```

### 4.2 添加属性元数据

在 `specPropertyMeta` 对象中添加元数据：

```typescript
新的属性_id: {
  id: '新的属性_id',
  group: 'basic',           // basic | thermal | mechanical | practical
  priority: 25,              // 数字越小越靠前
  comparable: true,          // 是否在对比表中高亮最优值
  highlightDirection: 'higher_is_better',  // higher_is_better | lower_is_better | neutral
},
```

### group 选择指南

| group | 用途 | 示例 |
|---|---|---|
| `basic` | 基本信息、外观、通用 | 材料类型、直径、密度、颜色、可打印性 |
| `thermal` | 热学相关 | HDT、Tg、熔融温度、打印温度 |
| `mechanical` | 力学性能 | 拉伸、弯曲、冲击、模量 |
| `practical` | 实用信息 | 优势、价格、环保 |

### priority 选择指南

| 范围 | 用途 | 说明 |
|---|---|---|
| 1-5 | 卡片预览 | Card 组件会显示 priority 最小的 3 个属性 |
| 6-20 | 重要属性 | 详情页靠前显示 |
| 21-50 | 次要属性 | 详情页靠后显示 |

## 5. Step 4：更新 spec-labels.ts（仅新属性时需要）

**文件：** `src/content/filaments/spec-labels.ts`

为每个新属性添加 **9 种语言** 的展示标签：

```typescript
新的属性_id: {
  'zh-cn': '新属性中文名',
  'zh-tw': '新屬性繁體名',
  'en': 'New Property Name',
  'ja': '新しいプロパティ',
  'ko': '새 속성 이름',
  'es': 'Nombre en español',
  'de': 'Deutscher Name',
  'fr': 'Nom français',
  'it': 'Nome italiano',
},
```

> 💡 **翻译技巧：** 英文名看厂商数据表的原文名。其他语言可用 ChatGPT/Claude 批量翻译，然后人工检查专业术语。

## 6. Step 5：验证

```bash
npm run build
```

如果 JSON 格式不对或用了未注册的 canonical ID，Astro 的 Zod schema 会报错。

常见错误：

| 错误信息 | 原因 | 修复 |
|---|---|---|
| `Unknown key in properties` | 用了不在 registry 中的 ID | 检查拼写或添加到 registry |
| `Expected number, got string` | 数值属性写成了字符串 | 把 `"value": "52.0"` 改为 `"value": 52.0` |
| `Missing required field` | 缺少必填字段 | 添加 `model`、`category` 等 |

## 7. 属性映射速查表

这是**最常用的参考表**。拿到新厂商数据时，先在这里查找对应关系。

### 基本信息 (basic)

| Canonical ID | 中文名 | 英文名 | 说明 |
|---|---|---|---|
| `material_type` | 材料类型 | Material Type | 如 "PLA（聚乳酸）" |
| `diameter` | 直径 | Diameter | 通常 1.75 mm |
| `density` | 密度 | Density | g/cm³ |
| `melt_flow_index` | 熔融指数 / 熔体流动速率 | Melt Flow Index / MFI / MFR | 厂商可能叫 MFI 或 MFR |
| `printability` | 可打印性 | Printability | 定性描述 |
| `moisture_sensitivity` | 吸湿性 / 是否易吸湿 | Moisture Sensitivity | 低/中/高 |
| `available_colors` | 颜色数量 | Available Colors | 如 "25+" |
| `spool_weight` | 卷重 / 净重 | Spool Weight / Net Weight | 如 "1 kg" |
| `eco_friendly` | 环保性 | Eco-Friendly | 如 "可生物降解" |
| `advantage` | 优势 / 特点 | Advantage / Features | 品牌特有描述 |

### 热学性能 (thermal)

| Canonical ID | 中文名 | 英文名 | ⚠️ 注意 |
|---|---|---|---|
| `heat_deflection_temperature` | 热变形温度 | HDT / Heat Deflection Temperature | **不是**维卡软化温度！ |
| `glass_transition_temperature` | 玻璃化转变温度 | Tg / Glass Transition Temperature | 变软但未熔 |
| `vicat_softening_temperature` | 维卡软化温度 | Vicat Softening Temperature | **不是**热变形温度！ |
| `melting_temperature` | 熔融温度 | Tm / Melting Temperature | 真正熔化 |
| `crystallization_temperature` | 结晶温度 | Tc / Crystallization Temperature | 冷却时结晶 |
| `print_temperature` | 打印温度 | Print Temperature / Nozzle Temperature | 来自部分厂商的简单数据表 |
| `bed_temperature` | 热床温度 | Bed Temperature | 来自部分厂商的简单数据表 |

### 力学性能 - XY 方向 (mechanical)

| Canonical ID | 中文名 | 英文名 | ⚠️ 注意 |
|---|---|---|---|
| `tensile_strength_xy` | 抗拉强度（XY）/ 拉伸强度（XY） | Tensile Strength (XY) | "抗拉"和"拉伸"是同一个东西 |
| `flexural_strength_xy` | 弯曲强度（XY） | Flexural Strength (XY) | |
| `flexural_modulus_xy` | 弯曲模量（XY） | Flexural Modulus (XY) | **不是**杨氏模量！ |
| `youngs_modulus_xy` | 杨氏模量（XY）/ 拉伸模量（XY） | Young's Modulus (XY) / Tensile Modulus (XY) | **不是**弯曲模量！ |
| `elongation_at_break_xy` | 断裂伸长率（XY） | Elongation at Break (XY) | |
| `impact_strength_xy` | 冲击强度（XY） | Impact Strength (XY) | **注意 test_method 字段** |

### 力学性能 - Z 方向 (mechanical)

| Canonical ID | 中文名 | 英文名 |
|---|---|---|
| `tensile_strength_z` | 抗拉强度（Z） | Tensile Strength (Z) |
| `flexural_strength_z` | 弯曲强度（Z） | Flexural Strength (Z) |
| `flexural_modulus_z` | 弯曲模量（Z） | Flexural Modulus (Z) |
| `elongation_at_break_z` | 断裂伸长率（Z） | Elongation at Break (Z) |
| `impact_strength_z` | 冲击强度（Z） | Impact Strength (Z) |

### 非方向性 (mechanical)

| Canonical ID | 中文名 | 英文名 | 何时使用 |
|---|---|---|---|
| `tensile_strength` | 抗拉强度 | Tensile Strength | 厂商未标注 XY/Z 方向时 |
| `elongation_at_break` | 断裂伸长率 | Elongation at Break | 厂商未标注 XY/Z 方向时 |
| `toughness` | 韧性 | Toughness | 厂商以 kJ/㎡ 为单位给出韧性/冲击韧性时 |
| `layer_adhesion` | 层间结合强度 | Layer Adhesion | 厂商给出层间结合强度时 |

### ⚠️ 冲击强度的特殊处理

冲击强度是最容易混淆的属性。不同厂商用不同测试方法，**数值不可直接对比**。

在 JSON 中用 `test_method` 字段标记：

```json
"impact_strength_xy": {
  "value": 5.51,
  "unit": "kJ/m²",
  "standard": "ASTM D256",
  "test_method": "notched_izod"
}
```

| test_method 值 | 中文名 | 英文名 | 常见标准 |
|---|---|---|---|
| `"notched_izod"` | 缺口悬臂梁冲击 / 缺口冲击强度 | Notched Izod Impact | ASTM D256 |
| `"charpy"` | 简支梁冲击 | Charpy Impact | ISO 179 |
| `"izod"` | 悬臂梁冲击（无缺口） | Izod Impact (unnotched) | ASTM D256 |
| `"charpy_notched"` | 缺口简支梁冲击 | Notched Charpy Impact | ISO 179 |

对比页面会自动显示测试方法差异的提示标记。

## 8. 常见问题

### Q：厂商叫 "拉伸强度"，系统中是 "抗拉强度"，怎么办？

**不需要改代码。** 它们映射到同一个 canonical ID `tensile_strength_xy`。展示标签在 `spec-labels.ts` 中统一定义，你只需要在 JSON 里用 canonical ID 即可。

### Q：厂商的数据表里没有 XY/Z 方向区分？

使用非方向性版本：`tensile_strength`、`elongation_at_break`。

### Q：厂商同时给了 "热变形温度" 和 "维卡软化温度"？

这是两个不同的属性，都填上：

```json
"heat_deflection_temperature": {
  "value": 53,
  "unit": "°C",
  "condition": "0.45 MPa"
},
"vicat_softening_temperature": {
  "value": 63,
  "unit": "°C"
}
```

### Q：厂商给了 "杨氏模量"，和已有的 "弯曲模量" 是一回事吗？

**不是！** 它们是完全不同的指标：

| 指标 | 含义 | canonical ID |
|---|---|---|
| 杨氏模量 / Young's Modulus | 拉伸刚度 | `youngs_modulus_xy` |
| 弯曲模量 / Flexural Modulus | 弯曲刚度 | `flexural_modulus_xy` |

### Q：厂商的数据表缺少某些属性（如没有拉伸强度），能从其他品牌或通用数据中估算填入吗？

**绝对不行。** 每个属性都必须有该品牌/型号的官方数据支撑。没有官方数据就不填。前端会自动显示 "—"。

### Q：某个属性只有这个厂商有，其他厂商没有？

**直接填写即可。** 对比页面会自动处理缺失情况——缺少的厂商显示 "—"。

### Q：需要添加全新的属性（速查表中没有的）？

按照 Step 3 和 Step 4 的步骤，更新 `spec-registry.ts` 和 `spec-labels.ts`，然后在 JSON 中使用新的 canonical ID。

### Q：需要更新页面组件吗？

**不需要。** 页面组件（列表页、详情页、对比页、CompareWidget）都是动态渲染的，会自动适配新增的属性和耗材。
