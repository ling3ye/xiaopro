## 项目背景

LINGFLUX 网站原本专注于硬件开发（开发板、模块、实验、方案、文章），需要新增一个 **3D打印** 栏目，内容涵盖：

1. **3D打印机目录** — 产品参数展示、参数对比、购买外链
2. **3D打印耗材目录** — 产品参数展示、参数对比、购买外链
3. **自设计3D模型分享** — MakerWorld 下载链接（赚积分）、打印设置、关联方案

### 为什么选方案 D

讨论过程中评估了四个方案：

| 方案 | 描述 | 优缺点 |
|------|------|--------|
| A | 作为 Article 的新 domain | 最轻量但身份模糊，没有独立导航 |
| B | 新建一个 `prints` collection | 所有东西塞一起，参数对比难做 |
| C | 完整镜像 Printers/Filaments/Models 三个独立生态 | 最完整但过度设计 |
| **D** | **镜像 boards/modules 模式 + 通用对比组件** | **平衡复用与扩展** |

方案 D 的核心思路：
- `printers` 镜像 `boards`（JSON 产品目录）
- `filaments` 镜像 `modules`（JSON 产品目录，按分类分组）
- `models` 类似 `solutions`（Markdown 内容，可跨集合引用）
- 通用对比组件一次开发，printers 和 filaments 复用

## 核心组件

### CompareWidget.astro（参数对比）

**版本迭代**：v1 → v2 → v3（当前）

**v1**：基础弹窗对比，仅列表页可用，选择不跨页面保持

**v2**：新增 localStorage 持久化，但使用 Astro 默认 `<script>` 导致跨页面同步失败

**v3（当前）**：使用 `is:inline` 确保每次页面加载都执行

核心特性：
- localStorage 持久化选择（跨页面保持）
- 列表页复选框 + 详情页按钮 双入口
- 底部浮动栏自动显示已选项
- 快速弹窗预览 + 跳转独立对比页面
- 最多 4 项同时对比

### AffiliateLink.astro（导购外链）

三种类型：
- `affiliate` — 赞助链接（黄色按钮，带「赞助链接」标签）
- `makerworld` — MakerWorld 下载（绿色按钮，带下载图标）
- `official` — 官方链接（普通样式）

支持三种变体：`button`、`card`、`inline`

## 遇到的问题与解决方案

### 问题 1：跨页面复选框不同步

**现象**：在耗材详情页点击「加入对比」后，回到列表页对应的复选框没有自动勾选。

**根因分析**：

1. **STORAGE_KEY 获取方式错误** — v2 版本通过 `document.getElementById('compare-bar')?.dataset.type` 获取集合类型，但详情页使用 `showBar={false}` 导致 bar 不渲染，key 回退为默认值 `'printers'`，与列表页的 key 不一致
2. **Astro `<script>` 的 bundle/dedupe 行为** — Astro 默认的 `<script>` 会被打包、去重、延迟执行。页面导航时脚本可能不会重新执行

**解决方案**：

1. **用隐藏元素传递数据** — 新增 `.compare-host[data-collection-type]` 元素（始终渲染），脚本从这里读取集合类型，不再依赖 `#compare-bar`
2. **使用 `is:inline`** — 脚本直接内联到 HTML，不经过 Astro 打包流程，每次页面加载 100% 执行
3. **IIFE 包裹** — `(function(){...})()` 确保每次都是全新执行上下文
4. **`requestAnimationFrame`** — 确保 DOM 完全渲染后再操作 checkboxes

```javascript
// 修复前（v2 - 可能不执行）
<script>
  const STORAGE_KEY = `compare_${document.getElementById('compare-bar')?.dataset.type || 'printers'}`;
  ...
</script>

// 修复后（v3 - 必定执行）
<script is:inline>
(function() {
  var hostEl = document.querySelector('.compare-host');
  var STORAGE_KEY = 'compare_' + (hostEl ? hostEl.getAttribute('data-collection-type') : 'printers');
  ...
  // 同步 checkboxes
  document.querySelectorAll('.compare-checkbox').forEach(function(cb) {
    if (selectedItems.indexOf(cb.dataset.compareId) !== -1) {
      cb.checked = true;
    }
  });
  requestAnimationFrame(updateBar);
})();
</script>
```

### 问题 2：详情页无对比反馈

**现象**：在耗材详情页点击「加入对比」后，页面没有任何视觉反馈，无法进入对比页面。

**根因**：`showBar={false}` 导致浮动对比栏完全不渲染。详情页的 CompareWidget 实例虽然有 `detail-add-compare` 按钮逻辑，但没有浮动栏来展示已选状态。

**解决方案**：

移除 `showBar` prop，改为单一实例模式。列表页和详情页共享同一个 CompareWidget，浮动栏始终渲染（由 JS 根据已选项数量控制显示隐藏）：

```astro
<!-- 修复前：两个实例，详情页不显示 bar -->
{isListPage && <CompareWidget collectionType="printers" locale={locale} />}
{!isListPage && <CompareWidget collectionType="printers" locale={locale} showBar={false} />}

<!-- 修复后：单一实例，bar 由 JS 控制 -->
<CompareWidget collectionType="printers" locale={locale} />
```

### 问题 3：Astro TypeScript 语法在 is:inline 中不支持

**现象**：`<script is:inline>` 中使用 TypeScript 语法（如 `as HTMLInputElement`、`Record<string, any>`）会报错。

**根因**：`is:inline` 脚本不经过 Astro 的 TypeScript 编译流程，直接输出原始代码。

**解决方案**：将所有 TypeScript 语法改为纯 JavaScript：
- `as HTMLInputElement` → 删除（不需要类型断言）
- `Record<string, any>` → 普通对象 `{}`
- `const` / `let` + 类型注解 → `var`
- 箭头函数 `() =>` → `function(){}`
- `includes()` → `indexOf() !== -1`（更兼容）
