私はOpenClawがブラウザを操作でき、システムファイルを操作できるのが気に入っています。もちろん、これには多くのセキュリティ問題が伴うため、注意して使用する必要があります。この小さなロブスターをうまく飼育するには、設定方法を学ぶために時間をかける必要があります。

## OpenClawのインストールを始めましょう

私はmacOSを使用していますが、これはロブスター飼育に最適なシステムです。なぜなら、作者がmacOSで開発したため、問題が最も少ないからです。その他のWindows、Raspberry PI OS、Linuxでは少し異なる場合があります。しかし、このプロジェクトには多くの貢献者がいるため、大きな問題もすぐに解決できます。

私の見解では、高い順から良い順は次のようになります：

macOS > Linux > Windows > others

まず、公式ウェブサイトでさまざまなインストール方法を確認してください：https://openclaw.ai/

### macOSでの高速インストール

macOSでは、公式のAPPダウンロードが提供されていますが、なぜか私の環境ではAPPをインストールできませんでした。1行の自動インストールコマンド`curl -fsSL https://openclaw.ai/install.sh | bash`を使用しても、毎回成功するわけではありませんでした。そのため、npmを使用したインストール方法を選択しました。コマンドラインで以下を入力します：

```bash
# Install OpenClaw
npm i -g openclaw
```

インストールの完了を辛抱強く待ってから、以下を入力し続けます：

```bash
# Meet your lobster | OpenClawの設定
openclaw onboard
```

その後、設定ウィザードのインターフェースが表示されます。時間をかけて確認しながら、APIを段階的に設定してください。一般的なプロセスは以下の通りです：



### 1. これは使用シナリオについて尋ねています：

- **Personal（個人使用）**：このMac Miniをあなた一人だけが使用する → **Yes**を選択
- **Shared/multi-user（マルチユーザー共有）**：複数のユーザーアカウントがこのマシンを共有し、追加の権限制御が必要 → Noを選択

**Yesを直接選択してください**、エンターキーを押して続行します。

![image-20260306103540426](https://img.lingflux.com/2026/03/f4a9f8ac970e447eadd09b4533d4c6c0.png)



### 2. これはどの**インストール設定方法**を選択するか尋ねています：

- **QuickStart（クイックスタート）**：まず高速でインストールし、詳細設定（APIキー、モデル選択など）は後でゆっくり設定
- **Manual（手動）**：今すぐすべての詳細を手動で段階的に設定

------

**QuickStartを選択することをお勧めします**。理由：

- まず動かして、後で`openclaw config set`で徐々に調整できる
- 時間と労力を節約できる



![image-20260306104637473](https://img.lingflux.com/2026/03/e3a2171975e988818fbaf4a59195d72d.png)



### 3. 重要：OpenClawの脳を駆動する**どのAIモデルプロバイダー**を選択するか。

簡単に言うと：**OpenClaw自体はAIを提供せず、AIサービスプロバイダーに接続する必要があります**。

------

一般的なオプションの説明

| オプション                   | 説明                          |
| ---------------------- | ----------------------------- |
| **OpenAI**             | GPT-4oなど、最も主流             |
| **Google**             | Geminiシリーズ                   |
| **XAI (Grok)**         | マスク氏のAI                   |
| **Moonshot AI (Kimi)** | 国内、中国語のサポートが良好            |
| **Mistral AI**         | 欧州のオープンソースモデル                  |
| **OpenRouter**         | 統合プラットフォーム、1つのキーで複数モデル |
| **Qianfan**            | 百度文心                      |
| **Volcano Engine**     | バイトダンス火山エンジン              |
| **Hugging Face**       | オープンソースモデルプラットフォーム                  |

ここでの選択は非常に重要です。オンラインで見ると、国内モデルを選択しても、国内モデルには国内版と海外版があるため、このリストから国内モデルプロバイダー（Qwen、Z.AIなど）を直接選択すると、モデルのAPIエンドポイントURLが異なる場合があるからです。

そのため、`Custom Provider`を選択することをお勧めします。エンターキーを押してください。（これにより、モデルエンドポイントは自分で定義されるため、間違うことはありません）



![image-20260306104908514](https://img.lingflux.com/2026/03/f0f7b7cf1b38d23375a9c9d36a8abea8.png)



その後、API Base URLを入力するよう求められます。ここで対応するURLをコピーしてください。デモでは、先ほど確認してもらったアドレスをコピーしました：
https://open.bigmodel.cn/api/anthropic   （入力完了後、エンターキーを押してください）

![image-20260306105524813](https://img.lingflux.com/2026/03/1bcc60387a3ba29b530e1bb53be24bf8.png)

今すぐAPI KEYを入力するか尋ねられます。「Paste API key now」（今すぐ入力）を選択してエンターキーを押してください。

![](https://img.lingflux.com/2026/03/05a5eca5b5fa006180662a65d2ae9381.png)

ここでAPI KEYをコピーして貼り付け、エンターキーを押してください。

![image-20260306105827355](https://img.lingflux.com/2026/03/291d99669c65b615d719844ddf743cee.png)

次に、選択したAIサービスプロバイダーがどの**API形式**を使用しているか尋ねられます：

- **OpenAI-compatible**：API形式はOpenAIと同じで、ほとんどのモデルプロバイダーがこの形式をサポートしています（OpenRouter、Kimi、火山エンジン、Mistralなど）
- **Anthropic-compatible**：API形式はAnthropic（Claude）と同じ

私のURLはClaude Code互換なので、「Anthropic-compatible」を選択します。

![image-20260306110230234](https://img.lingflux.com/2026/03/b515ca0d12b745463dee0ae52c35b1ab.png)

その後、モデルIDを入力するよう求められます。これは、AIプロバイダーのAPIで指定されている名前を確認する必要があります。

デモではGLM-4.7を使用しています（実際にはGLM-5.0がサポートされています。モデル名に従って入力してください）、エンターキーを押してください。

![image-20260306110457127](https://img.lingflux.com/2026/03/403f089e93c1d87df6cad0021532f953.png)

少し待つと、API接続テストが実行されます。成功すると「Verification successful」と表示されます。おめでとうございます。設定プロセスで最も複雑なステップを完了しました。

![image-20260306110630506](https://img.lingflux.com/2026/03/3d695e4ca1951bb4247a8e87c66f1295.png)

その後、Endpoint IDを入力するよう求められます（このAIサービスプロバイダーに**一意の名前**を付けて、区別しやすくします）。デフォルト値のままで問題ありません。エンターキーを押してください。

その後、Model aliasを入力するよう求められます（モデルに**短いニックネーム**を付けて、切り替え時にタイプ量を減らします）。空白のままでも入力しても構いません。エンターキーを押してください。

これは、いくつかの**追加機能プラグイン**を設定するかどうかを尋ねています。それぞれに対応するAPIキーが必要です：

------

| プロンプト                           | 機能                        | 必要なもの             |
| ------------------------------ | --------------------------- | -------------------- |
| **GOOGLE_PLACES_API_KEY**      | 地図/場所検索（Googleマップ） | Google Cloud API Key |

| **OPENAI_API_KEY (whisper)**   | 音声認識                  | OpenAI API Key       |
| **ELEVENLABS_API_KEY**         | AI音声合成（テキスト読み上げ）   | ElevenLabs Key       |

どう扱いますか？ **今はすべてNo/スキップを選択すれば大丈夫です**。理由：

- これらはオプション機能で、基本的な使用には影響しない
- 対応するアカウントがなければ強制的に入力しても使用できない
- いつでも戻って設定できる

![image-20260306111518497](https://img.lingflux.com/2026/03/66c6830490fd101cd8bc45b7b3bf041c.png)

### 7. **Hooksは自動化トリガー**で、特定のイベントが発生したときに特定の操作を自動的に実行します。

| オプション                      | 目的                                         |
| ------------------------- | -------------------------------------------- |
| **Skip for now**          | すべてスキップ                                     |
| **boot-md**               | 起動時に特定の指示/プロンプトを自動的にロード                |
| **bootstrap-extra-files** | 起動時に追加ファイルを自動的にロード                       |
| **command-logger**        | すべての操作ログを記録                             |
| **session-memory**        | `/new`または`/reset`を送信するときに現在の会話メモリを自動的に保存 |

------

**`session-memory`のみをチェックすることをお勧めします**。これが最も実用的で、AIが以前の会話内容を覚えることができます。

その他は初回使用時にスキップし、慣れてから有効にしてください。

## ブラウザ操作とファイル操作権限の設定

注意：以下の操作はOpenClawの権限を昇格させます。関連する内容を注意深く読んで、学習を続けてください。

### OpenClawのブラウザ制御には2つのモードがあります

1. （スタンドアロンモード）OpenClawの組み込みブラウザを使用し、ローカルシステムから完全に分離され、独立したログイン情報を持っています。

2. （拡張モード）ローカルChromeブラウザを使用し、OpenClaw Browser Relay拡張機能をインストールして、ローカルのログイン情報を共有します。

>  この例では**拡張モード**を使用し、ローカルChromeブラウザを使用します



### OpenClawゲートウェイトークンKEYを見つける

OpenClawゲートウェイトークンKEYはパスワードのように扱い、大切に保管してください。

OpenClawゲートウェイトークンKEYを取得する方法は以下の2つあります：

#### 1. 設定ファイルで直接見つける

macOSの設定ファイルはユーザーディレクトリにあり、非表示ファイルです。非表示ファイルを表示して`.openclaw`フォルダを見つける必要があります。開いてopenclaw.jsonファイルを開き、「gateway」セクションでトークンを見つけます。

![image-20260306122344215](https://img.lingflux.com/2026/03/537f4f44ca08c446ef9ede6549ddb90d.png)

#### 2. Web UIで確認する（この方法が最も簡単）

「Overview」をクリックし、「Gateway Token」の下に表示されます。非常に簡単に確認できます。

![image-20260306122950056](https://img.lingflux.com/2026/03/54be336e2aaa6342c1289fc5370c270a.png)





### 1. Chromeブラウザ制御拡張機能をインストールする

Chromeの拡張機能マーケットプレイス（https://chromewebstore.google.com/）で「OpenClaw Browser Relay」を検索してください

（執筆時点ではv2.7バージョンに到達しています）し、インストールします。（注：現在多くのOpenClaw Browser拡張機能があるので、よく見て間違えないでください）

インストールが完了すると、トークンKEYを入力して検証するページが表示されます。ここにはOpenClawのゲートウェイトークンKEYを入力してください。AI API KEYではありません。間違えないでください。

設定が完了したら、この拡張機能をピン留めして表示すると、操作しやすくなります。（後で手動操作が必要なので、覚えておいてください）

![image-20260306114923386](https://img.lingflux.com/2026/03/9b0e0c0e5bc27812210d0b9886a1962d.png)



### 2. デベロッパーモードを有効にする

ブラウザでchrome://extensions/を入力してエンターキーを押し、拡張機能管理ページに移動します。ページの右上隅に「Developer mode（デベロッパーモード）」のトグルがあります。有効になっているか確認してください。有効になっていない場合は有効にしてください。



### 3. OpenClawでブラウザとファイル操作をサポートするように設定する

CLI設定を使用します。ターミナル（コマンドラインツール）を開き、以下のコマンドを入力します：

```bash
# openclawの権限をcodingレベルに昇格
openclaw config set tools.profile coding

# ブラウザ機能が有効になっていることを確認
openclaw config set browser.enabled true

# システムChromeに切り替え（これが「システムデフォルトChrome」の公式方法です）
openclaw config set browser.defaultProfile "chrome"

# allowlistをクリア（codingプロファイルが正しいファイルツールを自動的に公開するようにします。これが重要です！）
openclaw config set tools.allow '[]'

# 設定完了、openclawゲートウェイを再起動（必須）
openclaw gateway restart
```

以下のように表示されます：

![image-20260306113653129](https://img.lingflux.com/2026/03/02f6c0258cc846cf7699cd372bbb8a03.png)

すべて正常であればおめでとうございます...でも...少し怖いと感じませんか？ もし...

そのため、ファイル操作権限を「workspace」作業ディレクトリに制限する必要があります。この作業ディレクトリはどこにあるのでしょうか？ openclaw.json設定を確認するか、Web UIの「Agents」→「Overview」で確認できます。

![image-20260306125311134](https://img.lingflux.com/2026/03/6b0ba79477058ff2819ac69caca95fb3.png)



### 4. ファイル操作を作業ディレクトリのみに制限する

CLI設定を使用します。ターミナル（コマンドラインツール）を開き、以下のコマンドを入力します：

```bash
# ファイル操作を作業ディレクトリのみに設定
openclaw config set tools.fs.workspaceOnly true
```

その後、ゲートウェイを再起動します：

```bash
openclaw gateway restart
```

### **`tools.profile`** 権限

https://docs.openclaw.ai/tools#tool-profiles-base-allowlist

OpenClaw（2026.3.2および最近のバージョン）の**`tools.profile`**は、ツール権限の**基本許可リスト（base allowlist）**です。この設定項目は**セキュリティに大きく影響します**。新規インストールのデフォルトは`"messaging"`（2026.3.xからの主要なセキュリティアップグレード）ですが、旧バージョンで多くのユーザーが慣れていたbroad/codingプロファイルは、明示的に設定する必要があります。

### 権限の比較（主な違い一目瞭然）

| 次元                                            | minimal               | messaging                 | coding                                     | full                  |
| ----------------------------------------------- | --------------------- | ------------------------- | ------------------------------------------ | --------------------- |
| **ファイル操作** (fs.read/write/edit/apply_patch)   | ✗ 完全に禁止            | ✗ 禁止                    | ✓ 許可（fs.workspaceOnlyでさらに制限可能） | ✓ 許可                |
| **シェル実行** (exec/runtime/process)           | ✗ 禁止                | ✗ 禁止                    | ✓ 許可（approvals.execレビューを追加可能）         | ✓ 許可                |
| **ブラウザ** (browser)                            | ✗ 禁止                | ✗ 禁止                    | ✓ 許可                                     | ✓ 許可                |
| **メッセージ/セッション管理** (sessions_*, messaging group) | session_statusのみ | ✓ 完全にサポート                | ✓ 完全にサポート                                 | ✓ 完全にサポート            |
| **画像/メモリツール** (image, memory_*)             | ✗ 禁止                | ✗ 禁止                    | ✓ 部分的にサポート                                 | ✓ 完全にサポート            |
| **その他の高リスクツール** (cron, gateway, nodesなど)      | ✗ 禁止                | ✗ 禁止                    | ✗ ほとんど禁止（allowが必要）                   | ✓ 可能にオープン            |
| **新規インストールのデフォルト**                                  | ✗ デフォルトではない              | ✓ はい（2026.3.xの主要な変更） | ✗ 手動で設定が必要                             | ✗ 手動で設定が必要        |
| **推奨対象者**                                    | 最大限のセキュリティ、チャットのみ      | 一般ユーザー、初心者、チャットメイン  | 開発者、コード/ファイルヘビーユーザー                  | テスト、POC、モデルを信頼する場合 |
